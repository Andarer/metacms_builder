## Web Deployment

### GitHub Pages

Automatic deployment via GitHub Actions:

```bash
# Triggered automatically on push to main branch
# Builds web version and deploys to GitHub Pages
```

**Workflow**: `.github/workflows/deploy-pages.yml`

**Access**: https://andarer.github.io/metacms_builder/

### Custom Domain

1. Create CNAME file in `web/` directory:
```
yourdomain.com
```

2. Configure DNS:
```
A record: 185.199.108.153
A record: 185.199.109.153
A record: 185.199.110.153
A record: 185.199.111.153
```

3. Update GitHub Pages settings:
   - Repository → Settings → Pages
   - Custom domain: yourdomain.com

### Build for Web

```bash
# Development
flutter run -d web

# Production
flutter build web --release --web-renderer=html

# Web renderer options:
# --web-renderer=html    (better performance)
# --web-renderer=canvaskit (better compatibility)
```

## Mobile Deployment

### Android

#### Generate Signing Key

```bash
# Create keystore
keytool -genkey -v -keystore ~/key.jks \
  -keyalg RSA -keysize 2048 -validity 10000 -alias key

# On Windows:
keytool -genkey -v -keystore %USERPROFILE%\key.jks \
  -keyalg RSA -keysize 2048 -validity 10000 -alias key
```

#### Create Key Properties

`android/key.properties`:
```properties
storeFile=/path/to/key.jks
storePassword=your_password
keyPassword=your_key_password
keyAlias=key
```

#### Build Signed APK

```bash
# Debug APK
flutter build apk --debug

# Release APK
flutter build apk --release

# App Bundle (for Play Store)
flutter build appbundle --release
```

#### Google Play Store

1. Create Google Play Developer account
2. Create new application
3. Prepare assets:
   - Icon (512x512)
   - Screenshots (up to 8)
   - Feature graphic (1024x500)
   - Description and privacy policy

4. Upload APK or App Bundle
5. Configure store listing
6. Submit for review

### iOS

```bash
# Build for iOS
flutter build ios --release

# Open in Xcode for signing
open ios/Runner.xcworkspace
```

**Note**: iOS support coming in v2.0

## Docker Deployment

### Build Docker Image

```bash
# Build
docker build -t metacms-builder:latest .

# Tag for registry
docker tag metacms-builder:latest myregistry/metacms-builder:latest

# Push to registry
docker push myregistry/metacms-builder:latest
```

### Run with Docker

```bash
# Development
docker run -p 8080:80 metacms-builder:dev

# Production
docker run -d \
  --name metacms-builder \
  -p 80:80 \
  -p 443:443 \
  -e FLUTTER_ENV=production \
  metacms-builder:latest
```

### Docker Compose

```bash
# Start services
docker-compose up -d

# Stop services
docker-compose down

# View logs
docker-compose logs -f metacms_builder

# Rebuild
docker-compose up --build
```

### Production Docker Setup

`docker-compose.yml` for production:

```yaml
version: '3.8'

services:
  metacms:
    image: metacms-builder:latest
    container_name: metacms-builder
    ports:
      - "80:80"
      - "443:443"
    environment:
      - FLUTTER_ENV=production
      - LOG_LEVEL=info
    restart: always
    volumes:
      - ./data:/app/data
      - ./config:/app/config
    networks:
      - metacms-network

  nginx:
    image: nginx:alpine
    container_name: metacms-nginx
    ports:
      - "8000:80"
    volumes:
      - ./nginx.conf:/etc/nginx/conf.d/default.conf
    depends_on:
      - metacms
    networks:
      - metacms-network

networks:
  metacms-network:
    driver: bridge

volumes:
  data:
  config:
```

### Kubernetes Deployment

`k8s/deployment.yaml`:

```yaml
apiVersion: apps/v1
kind: Deployment
metadata:
  name: metacms-builder
spec:
  replicas: 3
  selector:
    matchLabels:
      app: metacms
  template:
    metadata:
      labels:
        app: metacms
    spec:
      containers:
      - name: metacms
        image: myregistry/metacms-builder:latest
        ports:
        - containerPort: 80
        env:
        - name: FLUTTER_ENV
          value: production
        resources:
          requests:
            memory: "256Mi"
            cpu: "250m"
          limits:
            memory: "512Mi"
            cpu: "500m"
```

## CI/CD Setup

### GitHub Actions

**Workflows**: `.github/workflows/`

1. **flutter-build.yml** - Build and test
2. **deploy-pages.yml** - Deploy to GitHub Pages

### Manual CI/CD

Using other providers:

#### GitLab CI

`.gitlab-ci.yml`:
```yaml
stages:
  - build
  - test
  - deploy

build:
  stage: build
  image: cirrusci/flutter:3.16.0
  script:
    - flutter pub get
    - flutter build web --release
  artifacts:
    paths:
      - build/web

test:
  stage: test
  image: cirrusci/flutter:3.16.0
  script:
    - flutter test
```

#### Jenkins

`Jenkinsfile`:
```groovy
pipeline {
  agent any
  
  stages {
    stage('Build') {
      steps {
        sh 'flutter pub get'
        sh 'flutter build web --release'
      }
    }
    
    stage('Test') {
      steps {
        sh 'flutter test'
      }
    }
    
    stage('Deploy') {
      steps {
        // Deploy steps
      }
    }
  }
}
```
